Ты senior performance engineer и React/Node.js architect.

В проекте переключение роли пользователя (Мастер ↔ Заказчик) выполняется заметно медленно.

Твоя задача — найти точную причину задержки и предложить конкретный фикс.

Анализируй полный путь выполнения:

1. Нажатие кнопки переключения роли.
2. Frontend состояние (React, Zustand).
3. API запрос.
4. Backend обработка.
5. Supabase запросы.
6. Повторная авторизация.
7. Перезагрузка данных.
8. Перерисовка интерфейса.

Проверь:

### Frontend

* Лишние re-render'ы.
* Повторные запросы.
* Повторная загрузка профиля.
* Полный remount приложения.
* Блокирующие useEffect.
* Неэффективные Zustand subscriptions.
* Повторный вызов Telegram SDK.
* Лишние await.
* Неоптимальные AnimatePresence анимации.
* Избыточные invalidate/refetch.

### Backend

* Лишние запросы к Supabase.
* Последовательные запросы вместо Promise.all.
* Неэффективные SELECT *.
* Медленные JOIN.
* Повторные проверки профиля.
* Лишние проверки ролей.
* Повторная валидация initData.

### Database

* Наличие индексов.
* Медленные RLS policies.
* Полное сканирование таблиц.
* Лишние UPDATE.
* RPC вместо обычных запросов.
* Избыточные триггеры.

### Telegram Mini App

* Повторная инициализация SDK.
* Повторное получение initData.
* Повторная авторизация после смены роли.

Для каждой найденной проблемы укажи:

* Где находится.
* Почему тормозит.
* Сколько времени потенциально теряется.
* Конкретный фикс.
* Оценка ускорения (%).

Отдельно ответь:

1. Что является главным bottleneck.
2. Можно ли сделать переключение роли мгновенным (<200 мс).
3. Нужно ли вообще обращаться к API или достаточно локального обновления Zustand.
4. Какие изменения дадут максимальный прирост за минимальное время разработки.
